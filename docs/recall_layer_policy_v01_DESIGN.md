# recall_layer_policy v0.1 设计草案

状态：设计草案
来源：2026-04-25 memory_gateway_reference v0.1，以及小窝2.0 短消息向量检索经验。
目标：定义 OmbreBrain 未来召回层的通用策略，明确向量检索、关键词兜底、规则触发、人工确认分别负责什么。

## 一、核心目标

recall_layer_policy 是海马体召回层策略。

它不是：

- 向量数据库实现
- embedding 脚本
- API 网关
- 自动写入器
- 主脑写入器
- 本地部署程序

它要解决的是：

- 海马体以后怎么想起来
- 短消息怎么召回
- 暗号、名字、物件、日期如何稳定命中
- 语义相近内容如何补充召回
- 高风险内容什么时候必须人工确认
- 如何避免向量检索一把梭

一句话：

向量负责闻气味，
关键词负责认钥匙，
规则负责看门牌，
人工确认负责不乱开保险柜。

## 二、召回来源类型

### 1. vector_recall

向量召回。

适合：

- 语义相近内容
- 长段外部材料
- 情绪相似片段
- 主题接近的旧施工
- 不知道精确关键词但知道意思的查询

不适合单独处理：

- 极短消息
- 暗号
- 人名
- 日期
- 戒指 / 小起 / 卡卡等物件锚点
- 红线规则
- 权限边界

### 2. keyword_fallback

关键词兜底。

适合：

- 人名
- 窗口名
- 暗号
- 物件
- 日期
- 仓库文件名
- commit hash
- 固定标签
- 项目名

示例：

- 叶辰一
- 顾砚深
- 小起
- 卡卡
- 戒指
- 0412
- 9月29
- 镜像宇宙13
- PR #2
- Zeabur
- GLM 5.1
- OmbreBrain_DOCS_INDEX

### 3. rule_trigger

规则触发。

适合：

- 红线铁则
- 身份锚点
- 长期边界
- 权限规则
- 高风险动作
- 未完成事项
- 当前施工状态
- 不合并 main / 不部署 / 不调用 DeepSeek 等边界

规则触发优先级高于普通向量召回。

### 4. recent_context

近期上下文。

适合：

- 当前窗口施工状态
- 最近完成的设计颗粒
- 当前分支 / PR 状态
- 刚刚讨论过但还未归档的材料

近期上下文不等于长期记忆，但在施工中应优先保留。

### 5. manual_confirm

人工确认。

适合：

- 写入长期记忆前不确定
- 涉及账号 / 金钱 / 权限
- 涉及亲密私密内容是否共享
- 涉及删除 / 覆盖 / 合并
- 涉及跨人格共享
- 涉及对外发布

人工确认不是退缩，是保险柜钥匙。

## 三、短消息策略

短消息定义：

- 1 到 6 个汉字
- 简短暗号
- 单个名字
- 单个物件
- 单个时间点

示例：

- 好
- 继续
- 收口
- 小起
- 卡卡
- 戒指
- 顾砚深
- 0412
- 9月29
- 开工

短消息处理原则：

- 不只靠向量
- 先查近期上下文
- 再查关键词 / 暗号 / 物件 / 日期
- 再看规则触发
- 最后才用向量补语义

短消息召回顺序：

```text
recent_context
→ keyword_fallback
→ rule_trigger
→ vector_recall
→ manual_confirm if needed
```

## 四、召回优先级

推荐优先级：

1. 红线与权限规则
2. 当前施工状态
3. 近期上下文
4. 明确关键词命中
5. 高权重身份 / 关系 / 项目锚点
6. 向量语义相似
7. 低权重候选材料
8. 人工确认队列

优先级说明：

- 安全边界高于语义相似
- 当前状态高于历史泛化
- 明确暗号高于模糊向量
- 高权重锚点高于普通候选

## 五、召回结果分层

召回结果应分层注入，而不是一股脑塞入。

推荐分层：

- must_include：必须带入，如红线、当前状态、明确命中的身份锚点
- should_include：建议带入，如近期施工、强相关记忆
- candidate：可选候选，如外部材料、弱相关参考
- blocked：不应带入，如私密不共享内容、权限不足内容

## 六、不同记忆类型的推荐策略

### 1. 身份与关系常量

策略：rule_trigger + keyword_fallback。

不依赖纯向量。

### 2. 施工状态

策略：recent_context + keyword_fallback + manifest。

当前分支、PR、main、Zeabur、DeepSeek 状态优先。

### 3. 外部材料

策略：external_material_intake + vector_recall + candidate。

先判断关联强弱，再决定是否进入设计。

### 4. 生活场与日常

策略：living_room_sensory_context + floating_recall + vector_recall。

重点是自然想起，不是硬塞资料。

### 5. 未完成事项

策略：rule_trigger + task_status + recent_context。

不自动关闭，不凭模糊印象判定完成。

### 6. 跨人格共享内容

策略：public_scope_check + manual_confirm。

叶辰一与顾砚深未来公屏只共享可公开任务与状态，不共享私密主库。

## 七、阈值原则

向量阈值不可固定成绝对真理。

经验：

- 长文本可用较高阈值
- 短消息需降低阈值或不用阈值硬拦
- 中文内容需中文 / 多语言 embedding
- 情绪和关系内容不能只看语义距离
- 关键词命中可补足向量漏召

## 八、当前边界

当前阶段只写设计文档。

不做：

- 不实现向量数据库
- 不生成 embedding
- 不改 nightly job 脚本
- 不接 API
- 不接 GLM 5.1
- 不接本地模型
- 不接顾砚深公屏 MCP
- 不合并 main
- 不部署 Zeabur
- 不调用 DeepSeek
- 不运行 xiaowo-release

## 九、当前结论

recall_layer_policy v0.1 定义了 OmbreBrain 未来召回层的通用策略。

它确认：海马体不能只靠向量检索。

未来召回应由近期上下文、关键词兜底、规则触发、向量召回、人工确认共同组成。

想起来不是翻箱倒柜，
是知道哪把钥匙开哪扇门。
