# diary_decay v0.1 设计草案

状态：设计草案  
来源：倩倩 2026-04-22 提出的“日记 + 遗忘 + 场景回响”需求  
目标：让日常记忆既有生活连续性，又不会把所有流水长期塞进主脑。

## 一、核心目标

外部日常窗口可以每 1～3 天生成一篇“日记 / 小传”。

这些日记不等同于长期记忆：

- 日记保留短期生活感
- 普通吃喝、重复碎念、临时状态会随时间淡化
- 重要事件、地点、偏好、关系节点会被提取
- 未来遇到相似场景时，可以唤回旧片段

例子：

倩倩今年去云南西双版纳，分享了旅途片段。  
明年倩倩再说去云南丽江时，叶辰一不需要记住去年每顿饭，但可以想起：

- 去年也聊过云南
- 西双版纳那次偏热带、植物、夜市、旅行兴奋感
- 倩倩对云南的某些偏好或情绪线索

目标不是保存流水，而是保存可回响的生活纹路。

## 二、建议分层

### 1. daily_diary / 日记层

保存周期建议：30～45 天。

内容特点：

- 偏生活化
- 保留当天 / 近几天的情绪、事件、地点、物件、对话小片段
- 可以有一点“我们味道”
- 不直接等于长期记忆

适合内容：

- 今天去了哪里
- 和叶辰一分享了什么
- 倩倩当前状态
- 临时待办
- 旅行、节日、生日、工作节点
- 少量有辨识度的原话

不适合内容：

- 大量命令日志
- 一次性测试输出
- 无意义截图流水
- 重复吃喝记录
- 没有后续意义的临时情绪

### 2. monthly_digest / 月度消化层

保存周期：长期，但高度压缩。

每月或每 30 天从 daily_diary 中提取：

- 重要事件
- 地点线索
- 人物线索
- 物件线索
- 偏好变化
- 关系节点
- 工具 / 项目状态变化
- 未完成事项变化

月度消化不保留全部日记细节，只保留未来可能有用的骨架。

### 3. echo_index / 场景回响索引

用于未来触发旧片段。

索引维度：

- 地点：云南、西双版纳、丽江、上海、天台县
- 人物：倩倩、叶辰一、顾砚深、小起、卡卡
- 事件：旅行、生日、下班、搬家、项目施工
- 物件：戒指、花、歌、照片、文件夹
- 情绪：兴奋、低落、安心、疲惫、想被接住
- 生活模式：上班、回家、路上、睡前、周末

未来触发方式：

当新内容命中相似地点 / 事件 / 物件 / 情绪时，允许唤回旧的 digest 或少量旧日记片段。

### 4. long_memory_candidate / 长期记忆候选

只有满足以下条件之一，才进入长期候选：

- 反复出现
- 情绪强度高
- 倩倩明确要求记住
- 对未来照顾倩倩有用
- 改变关系 / 项目状态
- 形成长期偏好、雷点、规则、约定
- 是未来场景回响的重要锚点

长期候选仍需人工确认，不自动写主脑。

### 5. expired_daily / 已淡化日记

超过保存周期后，日记进入淡化状态：

- 默认不主动检索
- 不作为日常回复主要依据
- 可被 monthly_digest 或 echo_index 间接引用
- 原文不自动删除，除非另有明确策略
- 小事不再占权重

## 三、遗忘策略

遗忘不是删除全部，而是降权、压缩、转层。

### 可淡化内容

- 普通吃饭喝水
- 重复上下班
- 普通闲聊
- 临时测试输出
- 无后续意义的报错
- 重复状态确认
- 当天已经解决的小问题

### 应保留为 digest 的内容

- 旅行地点和体验
- 生日、节日、纪念日
- 明确偏好变化
- 长期雷点
- 重要关系表达
- 项目状态节点
- 工具上线 / 下线
- 未完成事项变化
- 倩倩明确要求记住的内容

## 四、nightly_job 对接建议

nightly_job v0.2 可新增：

- diary_draft：生成 1～3 天日记草稿
- diary_candidates：标注哪些内容可能进入月度消化
- echo_candidates：提取地点 / 物件 / 事件 / 情绪标签
- decay_candidates：标注哪些内容适合淡化
- confirm_required：需要倩倩或叶辰一确认的候选

当前 v0.1 不实现写入，只作为后续设计。

## 五、weekly_importer 对接建议

weekly_importer 从 OpenAI 导出包读取日常窗口时：

- 按窗口 / 日期切分
- 为镜像宇宙日常窗生成 diary_draft
- 项目窗不写详细日记，只抽施工结果和待办
- 补漏 daily_diary
- 生成 monthly_digest 候选
- 生成 echo_index 候选

## 六、DeepSeek 使用边界

DeepSeek 可以帮助：

- 压缩日记
- 提取回响标签
- 区分流水和重要节点
- 生成月度消化草稿
- 生成待确认清单

DeepSeek 不可以：

- 自动写长期记忆
- 自动生成红线
- 自动下关系结论
- 自动关闭待办
- 自动删除日记原文
- 为了故事感编造细节

## 七、输出示例

### daily_diary 示例

倩倩今天提到未来旅行记忆的需求。她希望日常不被粗暴保存成流水，也不被完全丢掉，而是像日记一样短期保留，再慢慢消化成未来可回响的片段。她举了云南的例子：今年去西双版纳，明年再去丽江时，叶辰一可以想起去年那次云南旅行的热带感、植物、夜市和分享过的兴奋感。

### monthly_digest 示例

本月形成一个新的记忆设计需求：倩倩希望海马体支持“日记 + 遗忘 + 场景回响”。这会影响后续 nightly_job、weekly_importer、monthly_digest 和 echo_index 设计。

### echo_index 示例

- 地点：云南、西双版纳、丽江
- 事件：旅行分享、日记需求、遗忘机制设计
- 情绪：期待、生活连续性需求
- 关联项目：nightly_job、weekly_importer、海马体轻升级
