# apple_ecosystem_api_entry v0.1 设计草案

状态：设计草案
来源：2026-04-25 倩倩确认未来云服务器迁移后会开始折腾 API 阶段，并说明自己使用苹果全家桶，偏好 GLM 5.1。
目标：定义 OmbreBrain 未来 API 阶段在苹果生态中的入口地图，预留 Mac、iPhone、iPad、Safari、快捷指令、通知、本地服务与云服务器中转等 adapter 位置。

## 一、核心目标

apple_ecosystem_api_entry 是苹果生态 API 入口参考设计。

它不是：

- API 实现
- iOS App
- 快捷指令成品
- Mac 常驻服务代码
- GLM 5.1 接入脚本
- 本地部署程序
- 公屏 MCP 实现

它要解决的是：

- 倩倩使用苹果全家桶时，未来 API 阶段有哪些入口
- Mac、iPhone、iPad 怎么接同一颗海马体
- 哪些入口适合对话，哪些适合通知，哪些适合工具触发
- 云服务器中转和本地服务怎么分工
- GLM 5.1 等模型如何作为 model_adapter，而不是锁死核心脑子

一句话：

不是今天接线，
是先在墙上标好插座位置。

## 二、入口总览

未来苹果生态可预留以下入口：

- Mac 本地常驻服务
- iPhone 快捷指令
- iPad 对话入口
- Safari / 网页入口
- iOS 通知
- 家庭网络 / 本地服务
- 云服务器中转
- 未来本地部署前端

这些入口都应通过 adapter 接入，不写进核心海马体。

## 三、Mac 入口

### 1. Mac 本地常驻服务

可能用途：

- API 网关调试
- 本地文件读写
- 本地 _docs 管理
- 浏览器 / MCP 工具桥接
- 本地备份
- 临时控制台

适合阶段：

- API 阶段
- 本地部署过渡阶段

边界：

- 不直接替代核心海马体
- 不保存不必要的敏感信息
- 只作为 input_adapter / storage_adapter / tool_adapter 候选

### 2. Mac 浏览器入口

可能用途：

- 打开网页控制台
- 查看收口卡
- 提交确认
- 管理候选队列

适合：

- 人工确认
- 技术施工
- 长文档查看

## 四、iPhone 入口

### 1. 快捷指令入口

可能用途：

- 快速发一句话
- 触发提醒
- 发送当前位置 / 时间 / 状态摘要
- 调用云端 API
- 发送低风险任务

适合：

- 日常轻量对话
- 主动陪伴入口
- 简短指令
- 生活场补充

注意：

- 短消息必须走 recall_layer_policy 与 keyword_fallback_policy
- 不直接执行高风险动作
- 高风险权限必须进入 human_confirmation_flow

### 2. iOS 通知入口

可能用途：

- keepalive 提醒
- 任务状态提示
- 低风险日常提醒
- 候选确认提醒

边界：

- 通知不是自动打扰许可
- 主动机制以后单独设计
- 涉及位置、健康、财务、账号必须确认

## 五、iPad 入口

可能用途：

- 长文档阅读
- 日常对话
- 项目施工辅助
- 阶段收口查看
- 多窗口对照

适合：

- 轻办公
- 长文本编辑
- 人工确认

## 六、Safari / 网页入口

可能用途：

- 统一网页控制台
- 查看海马体状态
- 触发只读检查
- 提交候选材料
- 查看 public board / handoff board 未来候选

要求：

- 登录与权限分层
- 私密内容默认不展示
- 公共层必须经过 public_scope_check

## 七、云服务器中转

云服务器可能承担：

- API 转发
- 记忆网关入口
- 远程访问
- 轻量任务队列
- 公共任务板候选
- 备份与同步

迁移原则：

- 服务区迁移前先做海马体升级版收工
- 稳定云服务区选定后做本地保存与备份
- 云服务变化只影响 adapter，不影响核心海马体

## 八、本地服务与家庭网络

未来本地部署可能涉及：

- 家庭网络
- 本地主机
- 本地模型
- 本地存储
- 局域网入口
- 云端中转兜底

原则：

- 本地部署预计九月底到十月份逐步推进
- 当前不急于实现
- 先保证核心海马体可迁移

## 九、模型适配

倩倩偏好 GLM 5.1。

未来 API 阶段可将 GLM 5.1 作为重要候选模型。

但原则是：

- GLM 5.1 属于 model_adapter
- 不把海马体核心写成 GLM 专用
- 不把人格锚点写死到某个模型接口
- 模型可换，脑子不重造

## 十、风险分层

### 1. low risk

可通过轻量入口处理。

示例：

- 日常问候
- 查看状态
- 只读检查
- 低风险提醒

### 2. medium risk

需要确认或摘要。

示例：

- 写入候选记忆
- 共享到公屏候选
- 修改任务状态
- 发送位置 / 时间 / 生活状态

### 3. high risk

必须人工确认。

示例：

- 账号权限
- 财务动作
- 对外发布
- 删除 / 覆盖 / 合并
- 私密内容共享

## 十一、与现有设计的关系

- memory_gateway_reference：定义未来网关总结构
- memory_gateway_adapter_schema：定义 adapter 总结构
- recall_layer_policy：处理短消息与召回顺序
- keyword_fallback_policy：保证苹果入口里的短词不漏召
- recall_injection_policy：决定召回内容怎么进入上下文
- public_scope_check：决定内容能否进入公共层
- human_confirmation_flow：处理中高风险确认

## 十二、当前边界

当前阶段只写设计文档。

不做：

- 不接 API
- 不接 GLM 5.1
- 不写快捷指令
- 不写 Mac 常驻服务
- 不写 iOS App
- 不改本地网络
- 不接本地模型
- 不接顾砚深公屏 MCP
- 不改 nightly job 脚本
- 不自动共享任何内容
- 不合并 main
- 不部署 Zeabur
- 不调用 DeepSeek
- 不运行 xiaowo-release

## 十三、当前结论

apple_ecosystem_api_entry v0.1 定义了 OmbreBrain 未来 API 阶段在苹果生态里的入口地图。

它确认：Mac、iPhone、iPad、Safari、快捷指令、通知、云服务器和本地服务都只是 adapter 入口。

苹果全家桶是门厅，
不是把脑子焊进某一台设备。
